# STATUS — 회사 역사 AR 아카이브

## 현재 상태: 자유공간 AR 동작 확인 ✅ — 콘텐츠(사진/영상/설명) 채우기 단계

### 확정 방식 (오늘 최종)
**자유공간 WebXR AR** — AR을 켜면 공간에 빛나는 유물이 떠 있고, **걸어가서 화면 중앙 조준점에 맞추고 탭하면** 내용 팝업(사진·영상·설명).
- Z 플립6에서 정상 동작 확인 (카메라/배치/탭 발견/도감 진행 저장)
- 마커(액자 스캔) 방식은 시도했다가 **폐기** — 사용자가 원한 건 "공간에 떠 있고 걸어가 탭" 방식

### 핵심 동작
- `ar.html` — 자유공간 AR (A-Frame 1.6 + WebXR immersive-ar, ARCore)
- `admin.html` — 편집기: 위에서 본 **지도에 드래그로 배치** + 제목/연도/색/설명/사진/영상, 자동 저장
- `content.json` — **모든 기기 공유 데이터**. AR이 이걸 읽음. (현재 7개, 위치만 흩어둠, 내용/미디어 미입력)
- 진행도(도감)는 기기 localStorage에 저장, HUD ↻ 로 리셋

### 데이터/배포
- 저장소: https://github.com/nada880117-cmd/ar-archive (public)
- AR 주소: https://nada880117-cmd.github.io/ar-archive/ar.html
- 편집 주소: https://nada880117-cmd.github.io/ar-archive/admin.html
- ※ 캐시 때문에 테스트 시 주소 끝에 `?숫자` 붙여서 새로 열기

### 편집 → 배포(공유) 흐름
1. PC `admin.html`에서 위치 드래그 + 내용 입력
2. "내보내기" → `content.json` 다운로드
3. 그 파일을 프로젝트 폴더에 넣고 개발자(Claude)에게 알림 → 커밋 → 모든 기기 반영
   (다른 기기에서 최신본 받으려면 admin "공유본 불러오기")

### ⚠️ 미디어(사진/영상) 규칙 — 중요
- **PC 로컬 경로(`C:\...`)는 절대 안 됨.** 폰엔 그 파일이 없음.
- 방법: ① 파일을 프로젝트 폴더에 넣기 → 개발자가 `assets/`에 올림 → 편집기에 `assets/photos/x.jpg` / `assets/videos/x.mp4` 경로 입력
       ② 또는 공개 URL(`https://...`)
- 사진 → "사진 주소" 칸(여러 줄), 영상 → "🎬 영상 주소" 칸

### 내일 할 일 (이어서)
1. **사진/영상 파일 받기** → 사용자가 파일을 폴더에 넣고 "이거 ○○에 넣어줘" → `assets/`에 올리고 content.json 연결
2. 각 콘텐츠 **제목·연도·설명 채우기** (지금 "새 프레임" 4개 등 더미 상태)
3. AR에서 **위치 미세 조정** (현장 공간에 맞게)
4. 다 되면 최종 점검 + QR 인쇄
